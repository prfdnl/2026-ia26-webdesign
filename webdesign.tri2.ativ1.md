# Desenvolvimento de Interface Web (HTML/CSS) para controle diário de notas de uma turma

Este projeto tem como objetivo criar uma interface web utilizando HTML e CSS para um sistema de controle diário de notas de uma turma. A aplicação deve ter:

- `index.html` - Página de login para acesso ao sistema (aluno ou professor).
- Professores
  - `professor-cadastro.html` - Página de `perfil` do professor, para atualização de informações. 
  - `professor-dashboard.html` - Página principal para o professor, onde ele pode acessar as funcionalidades de cadastro de disciplinas, alunos e notas.
  - `professor-cadastro-disciplina.html` - Página para o professor cadastrar disciplinas e associar alunos.
  - `professor-cadastro-notas.html` - Página para o professor cadastrar notas dos alunos e ocorrências diárias.
- Alunos
  - `aluno-cadastro.html` - Página de `perfil` do aluno, para atualização de informações. 
  - `aluno-dashboard.html` - Página principal para o aluno, onde ele pode acessar suas notas e ocorrências diárias.
  - `aluno-consulta-notas.html` - Página para o aluno consultar suas notas e ocorrências diárias.

Cada página deve ter sua própria estrutura HTML e estilo CSS, garantindo uma interface amigável e funcional para os usuários. O projeto deve ser entregue no repositório `webdesign.tri2.ativ1` no GitHub, seguindo as orientações de entrega estabelecidas.

# Desenvolvimento em Sala de Aula

Durante as aulas, todas as páginas serão desenvolvidas em conjunto, com explicações detalhadas sobre a estrutura HTML e o uso de CSS para estilização. Os alunos terão a oportunidade de praticar a criação de páginas web, aplicando os conceitos aprendidos em cada etapa do desenvolvimento.

# Atividades Individuais

Os alunos deverão criar suas próprias versões das páginas, personalizando o design e a estrutura de acordo com suas preferências, enquanto mantêm a funcionalidade necessária para o sistema de controle diário de notas. O objetivo é incentivar a criatividade e a aplicação prática dos conhecimentos adquiridos durante as aulas.

> [!NOTE]
> Durante as aulas, tudo que for necessário manter padronizado, para que a aplicação funcione corretamente, será indicado. Os alunos podem/devem personalizar o design e a estrutura das páginas, desde que nomeclaturas e elementos essenciais sejam mantidos conforme as orientações fornecidas durante as aulas.
